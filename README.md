## 1. Общее описание

Проект представляет собой сетевую игру для двух игроков, реализованную на языке Java с использованием сокетов и графического интерфейса JavaFX. Программа позволяет создавать игровые комнаты, подключаться к ним по порту и проводить партии по правилам составления слов на поле 15x15.

## 2. Основные механики

- **Игровой процесс:** Игроки поочередно размещают буквы на поле, формируя слова по горизонтали или вертикали.
    
- **Ограничения:** На каждый ход отводится 90 секунд. Первый ход обязательно должен проходить через центральную клетку поля. Слова не могут содержать разрывов («дыр»).
    
- **Система очков:** Очки начисляются сервером на основе стоимости букв и бонусов клеток.
    
- **Условия завершения:** Игра оканчивается при опустошении пула букв (после 4-х финальных ходов), при сдаче одного из игроков или потере соединения.
    

## 3. Архитектура и классы модели

Программная логика базируется на следующих сущностях:

- **Game (Игра):** Контроллер, содержащий доску, мешок с фишками, список игроков и данные об активном игроке.
    
- **Board (Доска):** Двумерный массив 15x15, состоящий из объектов клеток.
    
- **BoardCell (Клетка доски):** Хранит тип клетки (бонусы) и установленную фишку.
    
- **Bag (Мешок):** Содержит фишки букв, доступные для выдачи игрокам.
    
- **User (Пользователь):** Хранит учетные данные (ID, имя, хэш пароля) и статистику (победы, поражения, количество игр).
    
- **Player (Игрок):** Связывает объект пользователя с текущим счетом и его набором фишек (7 штук).
    
- **Room (Комната):** Отвечает за сетевые параметры (порт, хост) и список подключенных пользователей.
    

**Перечисления (Enums):**

- **CellType:** Типы бонусов (x2/x3 буква, x2/x3 слово).
    
- **Tile:** Определение буквы и количества поинтов за неё.
    

## 4. Сетевое взаимодействие

- **Подключение:** Реализована возможность создания комнаты по порту или подключения к уже существующей из списка.
    
- **Отказоустойчивость:** Система поддерживает восстановление статуса игрока при переподключении. Если игрок покидает игру, предусмотрено время ожидания (10 минут для одиночного нахождения в комнате, 5 минут до очистки порта после выхода всех участников).
    
- **Игровые события:** Реализован механизм предложения ничьей (валиден 30 секунд) и возможность добровольной сдачи.
    

## 5. Технические особенности

- **Протокол:** Используется собственный протокол передачи данных через сокеты.
    
- **GUI:** Интерфейс включает элементы рисования (класс Graphics) для визуализации игровых элементов и обеспечения работы пошагового режима.
    
- **Многопоточность:** Применена для разделения сетевых операций и обновления интерфейса, что исключает зависание программы при ожидании ответа от сервера.
